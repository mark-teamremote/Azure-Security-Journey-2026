# Stage 2: Azure Security Engineering Labs

This stage is dedicated to Azure security engineering and aligns with `AZ-500`. The purpose is to move from general administration into security-focused implementation and validation.

## Objectives
- strengthen Azure security engineering capability
- build practical labs around identity, posture, secrets, and monitoring
- create portfolio assets that reflect real security tasks, not only exam topics

## Focus Areas
- Entra ID security and privileged access
- network security controls
- Defender for Cloud
- Key Vault and secret protection
- monitoring, alerting, and incident visibility
- governance, policy, and security posture

## Planned Lab Themes
- tenant and identity hardening
- RBAC review and privileged access boundaries
- secure Azure network design
- Defender for Cloud setup and findings review
- monitoring and alert response workflow

## Portfolio Outcome
This stage should demonstrate:
- security-first thinking in Azure
- ability to identify and reduce risk
- understanding of how monitoring, identity, and network controls work together
- readiness for security-focused support or consulting work

## Direction
As this stage grows, each lab should be documented with:
- business scenario
- security objective
- implementation steps
- validation results
- lessons learned
