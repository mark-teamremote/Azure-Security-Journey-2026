# Stage 3: Terraform and Infrastructure as Code

This stage focuses on building repeatable Azure environments with Infrastructure as Code. The goal is to extend operational and security knowledge into automated deployment patterns.

## Objectives
- learn Terraform in an Azure-focused context
- reduce manual deployment steps
- model secure and reusable infrastructure patterns
- support future consulting work with cleaner deployment workflows

## Planned Project Themes
- resource group and network baseline deployment
- reusable VM deployment modules
- secure network patterns
- identity-aware or policy-aware deployment structure

## Why This Stage Matters
Infrastructure as Code increases credibility in three ways:
- it shows repeatability
- it improves speed and consistency
- it makes architecture decisions easier to explain and audit

## Portfolio Outcome
Each Terraform project should show:
- what problem it solves
- what Azure resources it deploys
- what security or governance choices are built in
- how the code can be reused or adapted

## Direction
This stage will be most useful when tied directly to earlier Azure admin and security work, rather than treated as a separate learning track.
