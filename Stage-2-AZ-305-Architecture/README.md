# Stage 2: Azure Architecture and Solution Design

This stage is focused on Azure solution design and aligns with `AZ-305`. It is the bridge between hands-on administration and higher-value architecture work.

## Objectives
- move from operating Azure resources to designing them well
- understand service tradeoffs, resilience, governance, and cost decisions
- create architecture artifacts that are useful in client and interview conversations

## Focus Areas
- high availability and disaster recovery
- identity and governance at scale
- compute and storage design choices
- migration and modernization planning
- cost optimization and operational design

## Planned Project Themes
- highly available application architecture
- small business Azure landing zone
- cost-optimized migration design
- hybrid connectivity and resilience planning

## Portfolio Outcome
This stage should demonstrate:
- architecture thinking, not just implementation
- ability to explain tradeoffs clearly
- awareness of reliability, governance, and cost constraints
- readiness for `AZ-305` and architecture-oriented support work

## Direction
Each project in this stage should include:
- business scenario
- design goals
- architecture diagram
- service selection rationale
- resilience and security considerations
- lessons learned
